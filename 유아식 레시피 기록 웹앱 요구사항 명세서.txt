# 유아식 레시피 기록 웹앱 요구사항 명세서

## 1. 프로젝트 개요

### 1.1 목적
아이에게 만들어 준 유아식 레시피를 기록하고, 냉장고에 있는 재료를 기반으로 만들 수 있는 레시피를 빠르게 찾아볼 수 있는 웹 애플리케이션. 개인 기록용으로 시작하되, 다른 양육자들과 레시피를 **공유 / 스크랩**할 수 있는 커뮤니티 레시피북으로 확장된다.

### 1.2 핵심 가치
- **간편한 기록**: 부담 없이 빠르게 레시피를 저장할 수 있어야 함
- **재료 기반 검색**: "지금 있는 재료로 뭘 만들지?" 문제를 해결
- **공유 가능한 레시피북**: 다른 사용자의 공개 레시피를 내 레시피북으로 스크랩·복사해 나만의 방식으로 수정 가능

### 1.3 사용 대상
- **1차 사용자**: 아이 유아식을 기록하고 관리하려는 양육자 (본인 포함)
- **2차 사용자**: 다른 사람들의 유아식 레시피 아이디어를 얻고 싶은 양육자
- 로그인한 사용자는 개인 레시피북을 가지며, 각자 레시피를 공개/비공개로 관리
- 게스트(비로그인) 사용자도 공개 레시피는 둘러볼 수 있음 (단, 스크랩/저장은 불가)
- 모바일에서 주방에서 보며 조리할 가능성이 높음

## 2. 기능 요구사항

### 2.0 사용자 계정 및 인증

**FR-0-1. 회원가입 / 로그인**
- 이메일 + 비밀번호 가입을 기본으로 지원한다.
- 소셜 로그인(구글, 카카오)을 선택적으로 제공하여 가입 장벽을 낮춘다.
- 비밀번호 재설정(이메일 링크)을 지원한다.

**FR-0-2. 프로필**
- 닉네임, 프로필 이미지(선택), 자기소개(선택)를 설정할 수 있다.
- 닉네임은 공개 레시피의 작성자로 표시된다.
- 실명/이메일은 외부에 노출되지 않는다.

**FR-0-3. 게스트 모드 (둘러보기 전용)**
- 로그인 없이도 공개 레시피 탐색/검색/상세 조회는 가능하다.
- 게스트는 레시피 등록, 스크랩, 즐겨찾기 등 저장이 필요한 기능은 사용할 수 없다.
- 저장 기능 시도 시 로그인/회원가입 유도 모달을 띄운다.

**FR-0-4. 회원 탈퇴 / 데이터 삭제**
- 사용자는 언제든 계정을 삭제할 수 있다.
- 계정 삭제 시 본인의 비공개 레시피는 즉시 삭제된다.
- 공개 상태로 다른 사용자가 스크랩해 간 레시피는 "삭제된 사용자"로 익명 처리되거나 사용자가 삭제 정책을 선택할 수 있다 (설계 결정 필요).

### 2.1 레시피 관리 (CRUD)

**FR-1. 레시피 등록**
- 로그인한 사용자는 새 레시피를 등록할 수 있다.
- 필수 입력: 레시피 이름, 재료 목록
- 선택 입력: 조리 순서, 사진, 메모, 태그, 조리 시간, 분량
- 등록 시 **공개 / 비공개** 여부를 선택한다. (기본값: 비공개)

**FR-2. 레시피 조회**
- 사용자는 **내 레시피북** 탭에서 본인이 등록/스크랩한 레시피 목록을 확인할 수 있다.
- 사용자는 **탐색(Explore)** 탭에서 다른 사용자들의 공개 레시피를 확인할 수 있다.
- 목록에서는 이름, 대표 사진, 태그, (공개 레시피의 경우) 작성자 닉네임과 스크랩 수가 보인다.
- 개별 레시피 상세 화면에서 전체 내용을 확인할 수 있다.

**FR-3. 레시피 수정/삭제**
- 사용자는 본인이 **등록한** 레시피의 모든 항목을 수정할 수 있다.
- 사용자는 본인 레시피를 삭제할 수 있다. (삭제 전 확인 다이얼로그 필요)
- 공개 → 비공개, 비공개 → 공개로 상태를 언제든 전환할 수 있다.
- 다른 사용자의 원본 레시피는 수정할 수 없다. (스크랩해서 복사본으로 수정)

**FR-4. 즐겨찾기 / 별점**
- 사용자는 "아이가 잘 먹은 레시피"를 즐겨찾기(⭐)로 표시할 수 있다.
- 즐겨찾기는 본인 레시피북 범위에서만 의미가 있다.

### 2.2 재료 기반 레시피 추천

**FR-5. 보유 재료 입력**
- 사용자는 현재 가지고 있는 재료를 여러 개 입력할 수 있다.
- 입력 시 기존에 등록된 재료명을 자동완성으로 제안한다.

**FR-6. 레시피 추천 로직**
- 입력한 재료만으로 만들 수 있는 레시피를 우선 표시한다.
- "재료 1~2개만 더 있으면 가능한 레시피"도 부가적으로 표시한다.
- 각 레시피 옆에 "보유 재료 매칭률 %"를 표시한다.
- 정렬 기준: 매칭률 높은 순 → 즐겨찾기 → 최근 등록 순

**FR-7. 부족한 재료 표시**
- 추천 결과에서 부족한 재료가 무엇인지 명확히 표시한다.
- 추천 대상은 **내 레시피북 + 공개 레시피**를 모두 포함하며, 사용자는 범위를 "내 레시피만 / 전체" 중 선택할 수 있다.

### 2.2.1 레시피 탐색 및 공유 (커뮤니티)

**FR-7-1. 공개 레시피 탐색(Explore)**
- 로그인 여부와 무관하게 공개 레시피 목록을 둘러볼 수 있다.
- 인기순(스크랩 수), 최신순, 관련도순 정렬을 지원한다.
- 태그/재료/개월수 태그로 필터링할 수 있다.

**FR-7-2. 레시피 스크랩**
- 로그인한 사용자는 다른 사용자의 공개 레시피를 **스크랩**할 수 있다.
- 스크랩한 레시피는 내 레시피북의 "스크랩" 섹션에 저장된다.
- 스크랩은 원본 레시피의 참조(링크)로 유지되며, 원본이 수정되면 변경사항이 반영된다 (원본 작성자 표시 유지).
- 원본이 비공개 전환/삭제되면 스크랩에서 "원본 비공개/삭제됨"으로 표시된다.

**FR-7-3. 레시피 복사(포크)**
- 로그인한 사용자는 공개 레시피 또는 스크랩한 레시피를 **내 레시피로 복사**할 수 있다.
- 복사본은 원본과 독립적이며, 자유롭게 수정할 수 있다.
- 복사본에는 "원작: [원작자 닉네임]" 정보가 남는다 (크레딧 표시).
- 복사된 레시피를 다시 공개로 게시할 때는 원작 크레딧이 유지된다.

**FR-7-4. 작성자 페이지**
- 공개 레시피 작성자의 닉네임을 클릭하면 해당 사용자의 공개 레시피 목록을 볼 수 있다.
- 팔로우/댓글 등 소셜 기능은 **범위 외**로 한다 (확장 아이디어).

**FR-7-5. 신고 기능**
- 부적절한 공개 레시피(스팸, 유해 정보 등)를 신고할 수 있다.
- 신고 누적 시 운영자 검토 후 숨김/삭제 처리된다.

### 2.3 태그 시스템

**FR-8. 자유 태그**
- 사용자는 각 레시피에 자유롭게 태그를 붙일 수 있다. (예: `후기이유식`, `돌전`, `간식`, `단백질`, `국`)
- 태그 목록을 통해 필터링하여 조회할 수 있다.
- 개월수 단계는 별도 필드가 아니라 태그로 관리한다. (간단함 우선)

### 2.4 검색 및 필터

**FR-9. 검색**
- 레시피 이름, 재료명, 태그로 검색할 수 있다.
- 검색 범위 스코프: "내 레시피북 / 전체 공개 레시피" 선택 가능.
- 메모 내용은 본인 레시피 검색 시에만 대상이 된다 (비공개 메모 보호).

**FR-10. 필터링**
- 태그별 필터, 즐겨찾기 필터, 스크랩 여부 필터를 지원한다.

## 3. 비기능 요구사항

### 3.1 사용성
- 모바일 우선 (Mobile-First) 반응형 디자인
- 레시피 등록은 3단계 이내로 완료 가능
- 한 손으로 조작 가능한 UI

### 3.2 성능
- 레시피 목록 로딩: 1초 이내
- 재료 기반 추천 응답: 1초 이내 (수백 개 레시피 기준)

### 3.3 데이터 저장
- 다중 사용자 지원을 위해 **서버 기반 저장**이 기본이다.
- 사용자별 데이터는 격리되며, 비공개 레시피는 소유자 외에는 조회 불가해야 한다.
- 공개 레시피는 별도 인덱스를 통해 탐색 성능을 확보한다.
- 이미지/사진은 오브젝트 스토리지(S3 호환)에 저장하고 DB에는 URL만 기록한다.

### 3.4 데이터 백업
- 본인 레시피(비공개 포함) 전체를 JSON으로 내보낼 수 있다.
- JSON 파일 업로드로 복원 가능.
- 이미지 포함 여부 옵션 선택.

### 3.5 오프라인 지원
- PWA로 구성하여 **내 레시피북**은 오프라인에서 조회 가능.
- 오프라인에서 작성한 레시피는 온라인 복귀 시 자동 동기화된다.

### 3.6 보안 및 권한
- HTTPS 필수, 비밀번호는 bcrypt 등 단방향 해시 저장.
- 인증은 JWT 또는 세션 쿠키 (HttpOnly, Secure) 사용.
- API 모든 쓰기 작업은 소유자 검증 필수 (본인 레시피만 수정/삭제).
- 공개 레시피 작성자는 별도 검증된 필드로만 노출(닉네임).
- Rate limiting 적용 (스팸/어뷰징 방지).

### 3.7 개인정보 및 운영 정책
- 개인정보처리방침, 이용약관 페이지 필요.
- 스크랩/복사 기능과 관련된 저작권/크레딧 정책 명시.
- 만 14세 미만 가입 제한 (국내 기준).
- 운영자용 신고 관리 백오피스(또는 최소한의 관리 페이지) 필요.

### 3.8 기술 스택 제안
- **프런트엔드**: Next.js (App Router) + TypeScript
- **상태 관리**: Zustand 또는 React Query (서버 상태)
- **스타일**: Tailwind CSS
- **백엔드**:
  - 옵션 A — Supabase (Auth + Postgres + Storage를 한 번에, 빠른 MVP)
  - 옵션 B — Next.js API Routes + Prisma + Postgres + S3
- **인증**: Supabase Auth 또는 NextAuth.js
- **이미지 처리**: 업로드 시 리사이즈/압축 (sharp)
- **검색**: 초기에는 Postgres full-text, 규모 커지면 Meilisearch/Elasticsearch

## 4. 데이터 모델 (초안)

### User
| 필드 | 타입 | 설명 |
|------|------|------|
| id | UUID | 고유 식별자 |
| email | string | 로그인 이메일 (unique) |
| passwordHash | string | 해시된 비밀번호 (소셜만 쓰면 null) |
| nickname | string | 공개 닉네임 (unique) |
| profileImage | string | 프로필 이미지 URL (선택) |
| bio | string | 자기소개 (선택) |
| provider | string | "email" / "google" / "kakao" |
| createdAt | Date | 가입일 |

### Recipe
| 필드 | 타입 | 설명 |
|------|------|------|
| id | UUID | 고유 식별자 |
| ownerId | UUID (FK → User) | 소유자 |
| name | string | 레시피 이름 (필수) |
| ingredients | Ingredient[] | 재료 목록 (필수) |
| steps | string[] | 조리 순서 |
| photos | string[] | 이미지 URL 목록 |
| tags | string[] | 자유 태그 |
| cookingTime | number | 분 단위 |
| servings | string | 분량 (예: "2회분") |
| memo | string | 자유 메모 (비공개 필드, 공개 시 노출되지 않음) |
| isFavorite | boolean | 본인의 즐겨찾기 여부 |
| visibility | enum | `private` / `public` |
| sourceRecipeId | UUID (FK → Recipe, nullable) | 복사해 온 원본 ID (포크/스크랩일 경우) |
| sourceOwnerNickname | string | 원작자 닉네임 스냅샷 (크레딧용) |
| scrapCount | number | 다른 사용자가 스크랩한 누적 수 |
| createdAt | Date | 등록일 |
| updatedAt | Date | 수정일 |

### Ingredient
| 필드 | 타입 | 설명 |
|------|------|------|
| name | string | 재료명 (예: "당근") |
| amount | string | 수량 (예: "30g", "1/2개") |
| optional | boolean | 선택 재료 여부 |

### Scrap
| 필드 | 타입 | 설명 |
|------|------|------|
| id | UUID | 고유 식별자 |
| userId | UUID (FK → User) | 스크랩한 사용자 |
| recipeId | UUID (FK → Recipe) | 스크랩된 원본 레시피 |
| createdAt | Date | 스크랩 일시 |
| (unique) | (userId, recipeId) | 중복 스크랩 방지 |

### Report (신고)
| 필드 | 타입 | 설명 |
|------|------|------|
| id | UUID | 고유 식별자 |
| reporterId | UUID | 신고자 |
| recipeId | UUID | 신고된 레시피 |
| reason | string | 사유 |
| status | enum | `pending` / `resolved` / `dismissed` |
| createdAt | Date | 신고 일시 |

## 5. 화면 구성 (정보 구조)

1. **로그인 / 회원가입**
   - 이메일+비밀번호, 소셜 로그인, 게스트로 둘러보기
   - 비밀번호 재설정

2. **홈 / 탐색(Explore)**
   - 공개 레시피 피드 (인기순/최신순)
   - 태그, 재료, 개월수 태그 필터
   - 상단 검색바
   - 로그인/비로그인 모두 접근 가능

3. **내 레시피북** (로그인 필수)
   - 탭: `내가 만든` / `스크랩` / `즐겨찾기`
   - 검색, 태그 필터, 공개/비공개 필터
   - 우하단 FAB(+)로 신규 등록

4. **레시피 상세**
   - 사진 → 재료 → 조리 순서 → 메모(본인 레시피만 노출) → 태그
   - 액션 버튼: 
     - 본인 레시피: 수정 / 삭제 / 공개·비공개 전환 / 즐겨찾기
     - 타인 공개 레시피: 스크랩 / 내 레시피로 복사 / 신고
   - 공개 레시피에는 작성자 닉네임, 스크랩 수 표시

5. **레시피 등록/수정**
   - 이름, 재료 동적 추가, 조리 순서 동적 추가
   - 사진 업로드, 태그 입력 (자동완성)
   - 공개/비공개 토글

6. **재료로 찾기**
   - 보유 재료 입력 (칩 형태)
   - 검색 범위 스위치: `내 레시피만 / 전체`
   - 추천 레시피 목록 (매칭률·부족 재료 표시)

7. **작성자 페이지**
   - 닉네임, 프로필 이미지, 자기소개
   - 공개 레시피 목록

8. **설정 / 프로필**
   - 프로필 편집 (닉네임, 이미지, 자기소개)
   - 비밀번호 변경
   - 데이터 내보내기 / 가져오기
   - 태그 관리
   - 로그아웃 / 회원 탈퇴

## 6. 개발 우선순위

### Phase 1 — 개인 MVP (나 혼자 쓰기)
- 로컬 or 단일 계정 기반 레시피 CRUD
- 재료 기반 추천 (기본 매칭률)
- 태그 기능
- 검색/필터
- JSON 내보내기/가져오기
- 사진 업로드 (선택)

### Phase 2 — 계정 시스템 도입 (다중 사용자 기반 구축)
- 서버 + DB 구축, 데이터 마이그레이션
- 이메일 회원가입/로그인
- 사용자별 레시피 격리
- 프로필 (닉네임)
- 이미지 스토리지 연동

### Phase 3 — 커뮤니티 기능 (스크랩/복사)
- 레시피 공개/비공개 설정
- 탐색(Explore) 피드 + 인기순 정렬
- 레시피 스크랩
- 레시피 복사(포크) + 원작 크레딧
- 작성자 페이지
- 신고 기능 + 간단한 운영 페이지

### Phase 4 — 경험 개선
- 소셜 로그인 (구글/카카오)
- PWA 오프라인 지원
- 자동완성 개선, 검색 품질 개선
- 태그/재료 정규화

### Phase 5 — 확장 아이디어
- 주간 식단 계획 기능
- 아이 반응/알레르기 기록 일지
- 좋아요/댓글/팔로우 등 소셜 기능
- 장보기 리스트 자동 생성

## 7. 향후 확장 아이디어 (기록용)

- 아이 식사 일지(먹은 날짜/반응) 연동
- 장보기 리스트 자동 생성 (부족한 재료 기반)
- 레시피 공유 링크 (읽기 전용)
- OCR로 요리책 사진에서 자동 등록

## 8. 제약사항 및 가정

- 초기(Phase 1)는 개인용으로 빠르게 검증하고, Phase 2부터 다중 사용자로 확장한다.
- 다중 사용자 지원을 위해 모든 쓰기 API는 소유자 검증을 거친다.
- 비공개 레시피는 본인 외 누구에게도 노출되지 않는다 (운영자 포함, 신고 처리 시 최소 범위 조회).
- 의료/영양 조언을 제공하지 않으며, 알레르기 관련 경고는 사용자가 직접 태그로 관리한다.
- 이미지 크기는 업로드 시 2MB 이하로 자동 리사이즈/압축한다.
- 스크랩/복사 기능은 저작권 정책에 따라 "원작 크레딧 유지"를 강제하며, 상업적 이용은 금지한다.
- 만 14세 미만은 가입이 제한된다.
